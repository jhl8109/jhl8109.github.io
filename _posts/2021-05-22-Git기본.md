C++ 토이프로젝트를 하면서 다른 레포지토리에도 업로드를 했습니다. 다른 레포지토리에 업로드하고 나니 또 깃 블로그 업로드 하려고 하는데 자꾸 막혀서 지금 공부한 기본 내용을 정리해놓으려고 합니다.

> Git Base <br>

##### 기본 설정

- git config --global user.name "사용자 이름"
- git config --global user.email 이메일

--global 옵션으로 사용자 이름과 이메일 주소를 설정합니다. Git이 commit 할 때마다 정보를 사용하기 때문에 처음 commit 한 후에는 정보를 재입력할 필요가 없어서 편리합니다. 

####  Git init<br>

로컬 저장소를 최초에 생성합니다. <br>

혹은 로컬 저장소를 다시 초기화합니다<br>

Git의 버전 관리를 하기위한 정보를 저장하고 있습니다. 



따라서 .git 디렉토리가 삭제되면 모든 변경 내용은 소실될 수 있습니다.<br>

####  Git remote <br>

주로 git remote add, git remote remove, git remote -v를 사용합니다.<br>

git remote 는 원격저장소를 사용할 경우 자주 사용하게 된다. <br>

git remote add 는 원격 저장소에 올릴 파일을 추가할 때 사용합니다.

git remote remove 는 원격 저장소 주소를 삭제합니다. 

git remote -v 는 저장해놓은 원격저장소 주소를 보여줍니다.



#### Git add

git status(현재 상태를 확인함)와 주로 함께 사용합니다.

git status를 통해 변경사항이 있는 내용을 확인하고 git add 를 통해 commit 할 준비를 합니다. 



#### Git commit

서버에 올려서 push할 준비를 합니다.

-m 옵션을 통해서 설명이나 정보를 담을 수 있습니다.

ex) git commit -m "First Ver"



#### Git push

업로드 합니다.

ex) git push origin main



#### Git pull

ex) git pull origin main

깃에 업로드 되어있는 자료를 가져올 수 있습니다. pull은 이후에 설명할 clone 과 달리 폴더가 생성되지 않기 때문에 폴더를 미리 생성해놓거나 생성되어있는 폴더에서 git pull 하는 것이 좋다고 생각합니다.



#### Git clone

ex) git clone "주소"

git push 와 달리 레포지토리 전체를 가져온다는 느낌입니다. 레포지토리명 그대로 가져옵니다.
